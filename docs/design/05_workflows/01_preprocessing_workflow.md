# 预处理流程 (Data Preprocessing Workflow)

预处理流程的主要职责是将原始的脑电数据转换为标准化、可供深度学习模型直接读取的数据格式。

配置细节详见 [Data Manager 详细设计](../04_components/02_data_manager.md#2-data-preprocessor-数据预处理器)。

## 执行步骤

1. **配置解析**: 用户通过 CLI 命令启动预处理。系统首先读取 `preprocess.yml` 配置文件，解析其中定义的各项数据/联合预处理操作（如滤波、ICA 去伪影、滑窗切片等）及其顺序。
2. **数据懒加载与校验**: 系统查询数据库定位原始数据集物理路径。为防止 OOM，系统默认采用**基于被试的懒加载架构**：按被试 `.mat` 文件名自然排序，单次将其**单名被试的数据片段**读入内存，并快速校验并转置为 `(sessions, recordings, channels, samples)` 原型格式。
3. **分层执行转换**: 单被试数据依次通过声明的执行流：
   - **Data 流**: 纯特征清洗（滤波、重采样）。
   - **Label 流**: 标签平滑或处理。
   - **Joint 流**: 执行可能切断长片段或改变特征与标签维度的联合操作（如 Epoching / Sliding Window，切片产生的新分段将沿着 `recording` 维度展开堆叠）。
4. **追加持久化与元信息记录**: 经过 3 条主线处理完毕的特定被试数据块，将以增量追加形式（`append` 或 `np.memmap`）写入至统一规范好的 `.npy` 数据对象中（位于 `~/.uesf/data/preprocessed/<数据集名称>`），随后框架释放该被试对应内存。全局跑完后，系统将这组预处理数据集信息录入到数据库 `preprocessed_datasets` 表以供查询追溯。
